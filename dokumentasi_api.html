Dokumentasi API

Mau bikin bot Telegram, integrasi ke website, atau automasi order OTP? Kamu bisa pakai API ini. Tinggal pilih server, panggil endpoint, dan OTP langsung masuk — atau pakai webhook untuk terima OTP real-time tanpa polling.

1 Ambil API Key
2 Pilih Layanan
3 Order Nomor
4 Polling / Webhook
5 Selesai!

Quick Start

Belum pernah pakai API? Ikuti 3 langkah ini dan kamu bisa order OTP pertamamu dalam 2 menit:

1. Buka halaman API Key dan generate key baru
2. Copy perintah di bawah, ganti otpk_xxxx dengan key kamu:
curl -X POST "https://otpinstan.com/api/reseller/s1/order.php" \
  -H "X-Api-Key: otpk_xxxx" \
  -H "Content-Type: application/json" \
  -d '{"platform_id":21,"country_id":7}'
3. Kamu akan dapat nomor HP + order ID. Cek OTP-nya:
curl "https://otpinstan.com/api/reseller/s1/check.php?order_id=S1-1234567" \
  -H "X-Api-Key: otpk_xxxx"

Autentikasi

Setiap request harus menyertakan API key. Ada 3 cara (semuanya didukung):

Cara Contoh Catatan
Header X-Api-Key: otpk_xxxx Direkomendasikan — aman, tidak muncul di log
Bearer Token Authorization: Bearer otpk_xxxx Direkomendasikan — standar umum, cocok untuk SDK/Postman
Parameter ?api_key=otpk_xxxx Mudah tapi key bisa terekspos di log server

Belum punya API key? Buat di halaman API Key.

Tips: Polling check.php setiap 5 detik = 12 req/menit, masih jauh dari limit. Kalau kena rate limit, kamu akan dapat response 429 — tunggu 1 menit lalu coba lagi.

Pakai webhook? Polling jadi tidak perlu — OTP otomatis dikirim ke URL kamu saat masuk. Lihat section Webhook.

Masa Aktif Order

Setiap order punya batas waktu 20 menit sejak dibuat.

Order dibuat → status pending, timer mulai
SMS masuk → status berubah ke received, ambil OTP dari field otp
User cancel → status cancelled, saldo otomatis dikembalikan
20 menit habis → provider auto-cancel, saldo dikembalikan
Penting: Setelah order, segera masukkan nomor ke app tujuan (WhatsApp, Instagram, dll). Jangan order dulu baru cari app-nya — 20 menit sangat singkat! Kalau SMS tidak datang dalam 3-5 menit, coba resend.php.

Order baru bisa dibatalkan setelah 2 menit sejak dibuat (anti-abuse rule).

Server 1 Endpoints

Base: https://otpinstan.com/api/reseller/s1/

Server 1 punya banyak variasi produk per layanan (harga & stok berbeda). Order bisa pakai 2 cara: product_id spesifik, atau platform_id + country_id (sistem auto-pilih termurah yang ada stok).

PENTING — Jangan hardcode product_id:

product_id di Server 1 sering berubah — produk yang habis stok otomatis dihapus dari list. Kalau bot kamu hardcode product_id lama, suatu saat akan error "PRODUCT_NOT_FOUND".

REKOMENDASI: Pakai mode platform_id + country_id — sistem otomatis cari produk termurah yang masih ada stok. Bot kamu lebih reliable.

POST s1/order.php Buat order baru

Buat order di Server 1. Ada 2 cara order:

  1. Mode A (RECOMMENDED) — kirim platform_id + country_id. Sistem otomatis pilih produk termurah yang ada stok. Bot kamu lebih reliable karena gak perlu update product_id terus.
  2. Mode B (Advanced) — kirim product_id spesifik. Cuma pakai ini kalau kamu memang butuh variasi tertentu. Wajib refresh product_id dari services.php tiap order karena product_id sering berubah.
Parameter Tipe Keterangan
platform_id integer Mode A ID platform (Shopee, WhatsApp, Telegram, dll) — lihat di services.php
country_id integer Mode A ID negara, contoh: 7 (Indonesia)
product_id integer Mode B ID produk spesifik dari services.php (jangan hardcode!)
service_name string opsional Nama layanan untuk dicatat di history

Menerima application/x-www-form-urlencoded atau application/json body.

// ✅ MODE A (RECOMMENDED) — Auto-pilih termurah yang ada stok
POST https://otpinstan.com/api/reseller/s1/order.php
X-Api-Key: otpk_xxxx
Content-Type: application/json

{"platform_id": 21, "country_id": 7}

// ⚠️ MODE B (Advanced) — Pakai product_id spesifik
// HARUS refresh product_id dari /services.php tiap kali order
// karena product_id sering dihapus/diganti

{"product_id": 835426288}

Cara Bikin Bot Tahan Banting:
  1. User pilih layanan (misal WhatsApp Indonesia)
  2. Bot panggil services.php?country_id=7 sekali untuk dapat platform_id
  3. Cache platform_id (jarang berubah)
  4. Saat order, kirim {platform_id, country_id} — bukan product_id
  5. Bot kamu aman dari "PRODUCT_NOT_FOUND"
Setelah order berhasil:
  1. Segera masukkan nomor HP ke app tujuan
  2. Polling check.php atau pakai webhook untuk terima OTP
  3. Nomor aktif selama 20 menit — setelah itu auto-cancel dan saldo dikembalikan

Webhook (Real-Time OTP) NEW

Daripada polling check.php terus-terusan, kamu bisa daftarkan URL webhook untuk terima OTP real-time, instant, <1 detik. Cocok untuk bot Telegram berskala besar dan sistem otomatis.

Cara Aktifkan Webhook
  1. Buka halaman Webhook Settings
  2. Daftarkan URL endpoint yang akan menerima webhook (URL kamu harus publik & HTTPS)
  3. Copy Secret Key untuk verifikasi signature
  4. Klik Test Webhook untuk pastikan URL kamu jalan
  5. Selesai! Setiap OTP masuk akan otomatis di-push ke URL kamu

Format Payload Webhook

URL kamu akan menerima POST request dengan body JSON:

POST https://your-url.com/webhook
Content-Type: application/json
X-Signature: 8ac27c59dfbf7fcb...        // HMAC-SHA256 dari body
X-Webhook-ID: wh_6a140c2ac825c8...      // Unique ID per request
User-Agent: OTPInstan-Webhook/1.0

{
  "event": "otp.received",
  "order_id": "S1-1466556",
  "phone": "6285123570978",
  "otp_code": "957418",
  "sms_text": "Your verification code is 957418",
  "service": "Shopee",
  "country": 7,
  "timestamp": 1779704282
}

Event Types

Event Kapan Dikirim
otp.received OTP berhasil diterima dari provider
order.cancelled Order dibatalkan (manual atau auto)
order.expired Order kadaluarsa (20 menit habis)
test.webhook Test webhook (klik tombol di dashboard)

Verify Signature (Keamanan)

Setiap request webhook punya header X-Signature. Verify supaya yakin request dari kami, bukan attacker:

<?php
$secret = 'wh_xxxxx';  // Secret Key dari dashboard

$body      = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_SIGNATURE'] ?? '';
$expected  = hash_hmac('sha256', $body, $secret);

if (!hash_equals($expected, $signature)) {
    http_response_code(401);
    exit('Invalid signature');
}

// Valid! Proses payload
$data = json_decode($body, true);
if ($data['event'] === 'otp.received') {
    $orderId = $data['order_id'];
    $otpCode = $data['otp_code'];
    // ... kirim OTP ke user kamu di Telegram, dll
}

http_response_code(200);
echo 'OK';
const crypto = require('crypto');
const express = require('express');
const app = express();

const SECRET = 'wh_xxxxx';

app.post('/webhook', express.raw({type: 'application/json'}), (req, res) => {
  const body = req.body.toString();
  const signature = req.headers['x-signature'];
  const expected = crypto.createHmac('sha256', SECRET).update(body).digest('hex');

  if (signature !== expected) {
    return res.status(401).send('Invalid signature');
  }

  const data = JSON.parse(body);
  if (data.event === 'otp.received') {
    console.log(`OTP ${data.otp_code} untuk order ${data.order_id}`);
    // ... kirim OTP ke user di Telegram
  }

  res.status(200).send('OK');
});

app.listen(3000);
from flask import Flask, request
import hmac, hashlib

app = Flask(__name__)
SECRET = 'wh_xxxxx'

@app.route('/webhook', methods=['POST'])
def webhook():
    body = request.get_data()
    signature = request.headers.get('X-Signature', '')
    expected = hmac.new(SECRET.encode(), body, hashlib.sha256).hexdigest()

    if not hmac.compare_digest(signature, expected):
        return 'Invalid signature', 401

    data = request.json
    if data.get('event') == 'otp.received':
        print(f"OTP {data['otp_code']} untuk order {data['order_id']}")
        # ... kirim OTP ke user di Telegram

    return 'OK', 200

if __name__ == '__main__':
    app.run(port=3000)
Catatan Penting:
  • URL kamu harus respon HTTP 200-299 dalam 5 detik
  • Webhook auto-disable kalau gagal 20x berturut-turut
  • Polling tetap aktif sebagai backup — webhook hanya tambahan
  • Webhook bekerja untuk semua server (S1, S2, S5) — satu URL untuk semua
Monitor Webhook: Lihat statistik & log delivery di Webhook Logs — sukses/gagal, response time, payload, response body.

Handling Error

error_code Artinya Solusi
PRODUCT_NOT_FOUND Product ID Server 1 sudah dihapus provider Pakai platform_id + country_id (jangan hardcode product_id)
OUT_OF_STOCK Stok habis (Server 1) Coba platform_id lain atau tunggu beberapa menit
NO_NUMBERS Stok habis (Server 2 / 5) Coba server/negara lain
NO_PRICE Layanan tidak tersedia Cek services.php
LOW_BALANCE Saldo tidak cukup Deposit dulu
CANCEL_TOO_EARLY Belum 2 menit setelah order Tunggu sesuai wait_seconds
CONFLICT Resend gagal (Server 1) Hanya bisa resend setelah OTP pertama masuk
BAD_SERVICE Kode layanan tidak valid Ambil dari services.php
BAD_COUNTRY ID negara tidak valid Ambil dari countries.php

FAQ

Berapa lama nomor aktif setelah order?

20 menit. Setelah itu auto-cancel dan saldo dikembalikan otomatis.

Apa bedanya polling vs webhook?

Polling — kamu yang tanya server tiap beberapa detik via check.php. Sederhana, cocok untuk script kecil.
Webhook — server kami yang push OTP ke URL kamu begitu masuk. Instant (<1 detik), hemat resource. Cocok untuk bot besar.
Pakai keduanya — webhook untuk speed, polling sebagai backup kalau webhook gagal.

Kenapa cancel ditolak "CANCEL_TOO_EARLY"?

Order baru bisa dibatalkan setelah 2 menit. Rule ini cegah abuse refund instant. Cek field wait_seconds untuk tahu berapa detik lagi harus tunggu.

Kapan pakai Server 1 vs 2 vs 5?

Server 1 — sistem product_id per variasi, biasanya termurah, banyak pilihan harga per layanan.
Server 2 — pakai kode service singkat (wa, tg), punya operators.php untuk pilih provider spesifik.
Server 5 — mirip Server 2 tapi tanpa operators.php, lebih simpel.

Tips: Coba Server 1 dulu (harga termurah), kalau habis baru pindah ke 2 atau 5.

Bisa kirim JSON body untuk POST?

Ya! Semua endpoint POST menerima application/x-www-form-urlencoded dan application/json.

Bagaimana cara bikin bot Telegram OTP?

1. User pilih layanan → tampilkan inline keyboard dari services.php
2. User pilih negara → countries.php
3. Bot order → order.php, kirim nomor ke user
4. Pakai webhook untuk dapat OTP instant, atau polling check.php tiap 5 detik
5. OTP masuk → kirim ke user di chat
6. Timeout → auto cancel via cancel.php
