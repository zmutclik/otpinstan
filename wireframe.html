API Docs

Endpoint yang bisa dipanggil oleh client kamu

GET /{username}/getotp Order OTP baru

Membuat order OTP baru ke OTPInstan dan menyimpannya ke database.

Query Parameters
ParamTipeKeterangan
servicestring *wajibKode service: wa, tg, ig, dll
countryinteger *wajibID negara (Indonesia = 6)
Response
{
  "success": true,
  "order_id": "S5-123456789",
  "phone": "628123456789",
  "service": "wa",
  "country": 6,
  "price": 3200,
  "status": "pending"
}
GET /{username}/chekotp Cek status & ambil OTP

Cek apakah OTP sudah masuk untuk nomor HP tertentu. Selalu mengembalikan OTP terbaru — jika ada SMS baru masuk (resend), nilai otp otomatis terupdate ke yang terbaru.

Query Parameters
ParamTipeKeterangan
phonestring *wajibNomor HP, contoh: 628123456789
Response
// OTP belum masuk
{
  "success": true,
  "order_id": "S5-123456789",
  "phone": "628123456789",
  "status": "pending",
  "otp": null
}

// OTP masuk (atau sudah diperbarui jika ada resend)
{
  "success": true,
  "order_id": "S5-123456789",
  "phone": "628123456789",
  "status": "received",
  "otp": "445566",        // selalu OTP terbaru
  "otp_updated_at": "2026-07-01T14:34:02Z"
}
ℹ️ Catatan: Setiap SMS baru yang masuk ke nomor ini akan menimpa nilai otp sebelumnya. Client cukup polling endpoint ini — tidak perlu tahu berapa kali SMS masuk.
GET /{username}/status/{result}/{phone} Lapor hasil verifikasi

Client melaporkan apakah nomor HP berhasil digunakan untuk verifikasi atau tidak.

GET /john/status/gagal/628123456789

  "order_id": "S5-123456789"
